Introduction

This page documents the output of Canchek searches. For details of how to obtain a login and interact with the Canchek API, please see our Postman page.

Search Results

A Canchek search may return data from any of Canchek's seven databases, as listed below. Users may elect to search any combination of these databases. This election must be made in the web UI (there is no public API for this); selections are defined on a user by user basis, and apply to all forms of search (web UI, batch files, and this API). If a database is selected but has no search results, it will be included in the output JSON as an empty array. If a database is not selected, it will not be included in the output JSON at all. Common Fields

Several fields are common to one or more databases. These are documented here. Fields specific to a particular database are listed separately.
Field Description
falsePositive Indicates that this record has been flagged as a false positive. This designation is made by Canchek customers when a match is suggested but subsequently determined to not be relevant. In general, false positives can safely be ignored.
firstName Names are in general mixed-case but case standards may vary depending on source data. They may, and often do, contain characters outside the North American A-Z alphabet.
lastName See above.
name See above. Some databases differentiate last name from first name, others do not.
ref Most Canchek databases include a reference code that we use to uniquely identify each record. These codes may be managed by Canchek, third-party vendors, or government sources. The format varies from database to database. Reference codes will never be reassigned.

ceaseTradeOrderResult

This database lists individuals and entities recorded in the CSA Cease Trade Order system. The quality of this data is somewhat lower than in other databases, and for this reason false positives are more likely.
Field Description
exception TBD
failureToFile CTO was implemented due to a failure to meet CSA filing requirements.
issuedDate When the CTO was originally issued
persons The persons and/or entities named in the CTO. This may include many entries and is not structured.
effectiveJurisdictions The provinces and territories in which the CTO is effective. This field recognizes reciprocal CTO arrangements.
issuingJurisdictions The provinces and territories in which the CTO was issued.
companyName The name of the relevant company.
status The CTO's current status in the CSA CTO database.
type 'Trading BY' (the issue is with the named individuals or entity, they generally aren't allowed to trade anything) or 'Trading OF' (the issue is with the named issuing company, generally nobody is allowed to trade it).

domesticPoliticalPersonResult

This database lists all Canadian residents/citizens who qualify as a Politically Exposed Person. This data is maintained by Canchek Corp.
Field Description
photoUrl Link to a photo of the subject person. Various image formats may appear here. The quality of this link is not verified by Canchek Corp.
jurisdiction Area of influence (riding, municipality, tribe etc.)
role Job title or function (President, Minister, Chief etc.)

globalPoliticalPersonResult

This database lists worldwide (including Canadian) persons who qualify as Politically Exposed Persons. This data is maintained by a third-party vendor.
Field Description
birthDate Generally supplied in DD/MM/YYYY format, but this may vary.
text A free-format description of the individual's role (eg. President) and area of influence (eg. Spain).

insiderResult

This database lists insiders recorded on CSA's SEDI database. Note that the Canchek UI exposes additional fields from this data that are not available in this API.
Field Description
issuerName An insider may be related to one or more issuers. Issuers are generally Canadian private or public companies.
municipality Where the insider resides.

officeForeignAssetsControlResult

This database lists insiders recorded on the USA OFAC system. Many records from this database may also appear on Canchek's Canadian Sanctions database.
Field Description
aliases The name of known aliases may be provided.
placeOfBirth May be a specific location or a country.
programs The sanctions program under which this person or entity's name appears. These codes are maintained by OFAC.

ukResult

This database lists people or entities who appear on United Kingdom terrorist (or similar) watch lists. Many records from this database will overlap those of OFAC. This database is maintained by the British government.
Field Description
aliases The name, date of birth and place of birth of known aliases may be provided.
dateOfBirth Generally supplied in YYYY/MM/D format, but this may vary.
nationality Generally the commonly used name of the country.
otherInformation Generally a reference to UN sanctions.
regime Generally a reference to the sanctions program.

watchListResult

This database lists people or entities who appear on Canadian terrorist (or similar) watch lists. Many records from this database will overlap those of OFAC. This database is maintained by Canchek Corp.
Field Description
aliases The name, date of birth and place of birth of known aliases may be provided.
obligations Indicates the reporting requirements if a valid match is found.
placeOfBirth May be a specific location or a country.
regulations Regulations under which this person or entity's name appears.
sanctions Sanctions applicable to this individual or entity.

Errors

Errors encountered while processing a request will be returned with HTTP status code 200 and a JSON object describing the problem. The structure of this object is shown below. 

{
    "messages": [
        "Please enter both a first name and a last name."
    ]
}
